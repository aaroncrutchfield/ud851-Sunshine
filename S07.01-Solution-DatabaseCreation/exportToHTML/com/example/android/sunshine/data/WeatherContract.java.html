37   //      COMPLETED (3) Create a public static final String call COLUMN_DATE with the value "date" 
38           /* 
39            * The date column will store the UTC date that correlates to the local date for which 
40            * each particular weather row represents. For example, if you live in the Eastern 
41            * Standard Time (EST) time zone and you load weather data at 9:00 PM on September 23, 2016, 
42            * the UTC time stamp for that particular time would be 1474678800000 in milliseconds. 
43            * However, due to time zone offsets, it would already be September 24th, 2016 in the GMT 
44            * time zone when it is 9:00 PM on the 23rd in the EST time zone. In this example, the date 
45            * column would hold the date representing September 23rd at midnight in GMT time. 
46            * (1474588800000) 
47            * 
48            * The reason we store GMT time and not local time is because it is best practice to have a 
49            * "normalized", or standard when storing the date and adjust as necessary when 
50            * displaying the date. Normalizing the date also allows us an easy way to convert to 
51            * local time at midnight, as all we have to do is add a particular time zone's GMT 
52            * offset to this date to get local time at midnight on the appropriate date. 
53            */ 
54           public static final String COLUMN_DATE = "date"; 
55
